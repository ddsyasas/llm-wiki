# Code of Conduct

This project follows the **[Contributor Covenant 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/)** — the same code of conduct adopted by thousands of open-source projects across the ecosystem. The canonical text lives at the link above; this file summarizes what it means in practice.

## In short

Treat everyone with respect. Assume good faith. Disagree about ideas, not about people. Welcome newcomers — most of us were beginners once.

## What we expect

- **Be respectful in all communication** — issues, PRs, discussions, commit messages, code review comments.
- **Welcome people from any background** — experience level, identity, geography, language.
- **Assume good intent first**, ask for clarification before assuming hostility.
- **Accept constructive feedback gracefully**, both giving and receiving.
- **Focus on what's best for the project and its users**, not on winning an argument.

## What's not OK

- Personal attacks, insults, or derogatory comments about any contributor.
- Harassment in any form, public or private.
- Sharing someone else's private information without consent.
- Sustained disruption — derailing discussions, repeated off-topic posting, refusing to engage with feedback in good faith.
- Any other conduct a reasonable person would consider unprofessional in a collaborative open-source setting.

The full canonical list of unacceptable behaviors is in the [Covenant text](https://www.contributor-covenant.org/version/2/1/code_of_conduct/#our-standards).

## Where this applies

Anywhere project work happens publicly or that represents the project — GitHub issues, PRs, discussions, commit messages, code reviews, project Twitter / Discord / etc. if those exist later. Also covers behavior between contributors that's tied to project participation, even outside the above venues.

## Reporting

If you see or experience behavior that violates this code, report it:

- **Privately via email**: yasasdreamz@gmail.com
- **Privately via GitHub**: open a [private security advisory](https://github.com/ddsyasas/llm-wiki/security/advisories/new) — the same channel used for security issues. It's encrypted, only maintainers see it.

Reports are taken seriously and confidentially. Reporters won't be retaliated against, regardless of who the report is about.

## Enforcement

Maintainers (currently just [@ddsyasas](https://github.com/ddsyasas)) handle reports. The standard escalation path:

1. **Private conversation** — explain what behavior was a problem and why
2. **Public warning** if the behavior continues
3. **Temporary or permanent ban** from the project for repeated or severe violations

Severe violations (harassment, threats, doxing, etc.) skip steps 1-2 and go straight to a ban.

Decisions can be appealed by emailing yasasdreamz@gmail.com — a different perspective on the situation is welcome, though "I didn't break the rules" arguments after a clear violation will not change the outcome.
